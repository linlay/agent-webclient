# 计划事件与任务视图

## 当前状态
结构化计划由 `plan.create`、`plan.update` 和 `task.*` 事件驱动，状态保存在 `state.plan`、`planRuntimeByTaskId`、`taskItemsById` 等字段。浮动计划面板由 `PlanPanel` 渲染。

## 核心职责
- 将 plan create/update 归一为 `Plan`。
- 将 task start/complete/fail/cancel 更新为 task runtime 和 task meta。
- 在计划面板展示总进度、每个任务状态和运行耗时。
- 将 timeline node 与 taskId/taskGroupId/subAgentKey 绑定，便于按任务观察运行过程。

## 核心流程
`eventProcessorPlan.ts` 处理结构化计划事件并派发 `SET_PLAN`、`SET_PLAN_RUNTIME`、`SET_TASK_ITEM_META` 等命令。`PlanPanel` 读取全局状态，按任务状态计算进度、状态点和耗时，并提供展开/收起交互。

## 边界与非目标
- PlanPanel 不是 HITL plan decision 弹窗；后者属于 awaiting plan。
- 计划视图只展示后端事件，不在前端生成或修改 plan。
- 任务耗时基于前端收到的时间戳计算，仅用于观察。
