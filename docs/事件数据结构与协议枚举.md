# 事件数据结构与协议枚举

## 当前状态
前端事件类型集中在 `src/app/state/eventTypes.ts`，时间线节点、工具态、导航实体和 UI 资源文件分别在 `timelineTypes.ts`、`toolTypes.ts`、`navigationTypes.ts`、`uiTypes.ts` 中声明。前端只消费后端事件，不在本仓库定义后端协议事实源。

## 核心职责
- 为 `AgentEvent`、`TimelineNode`、`ActiveAwaiting`、`ToolState`、`PlanRuntime` 等渲染结构提供类型约束。
- 枚举 run、content、reasoning、planning、tool、awaiting、usage、source 等事件类型。
- 支撑 replay、live stream、debug panel 和右侧概览面板读取同一套字段。
- 为事件处理器提供最小可用的字段读取约定。

## 核心流程
传输层收到 SSE 或 WebSocket 帧后归一为 `AgentEvent`。事件处理器按 `event.type` 分派到 run、content、reasoning、planning、tool、plan、source 等处理函数，再生成 timeline node、tool state、artifact、plan runtime 等前端状态。

## 边界与非目标
- 事件类型文件只描述前端已消费字段，不承诺后端全量协议。
- 字段兼容处理应放在事件处理或数据读取工具中，不在 UI 组件里散落判断。
- 后端新增协议时，先确认实际事件样例，再补前端类型和测试。
