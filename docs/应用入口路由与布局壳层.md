# 应用入口路由与布局壳层

## 当前状态
`agent-webclient` 是单页 React 应用，入口由 `src/app/index.tsx` 挂载，`src/app/App.tsx` 负责装配路由、Ant Design 主题、应用状态上下文和全局运行时。主对话体验由 `AppShell`、`CopilotShell`、`AgentChatShell`、左右侧栏、底部 dock 与模态层组合。

## 核心职责
- 建立根路由、独立管理页路由和默认对话路由。
- 在应用顶层注入 `AppProvider`、主题、国际化和全局样式。
- 区分实时对话页面与独立管理页面，避免管理页误走实时传输模式。
- 承载全局 overlay、modal、dock、sidebar 等壳层 UI。

## 核心流程
浏览器加载 `public/index.html` 后进入 `src/app/index.tsx`，再由 `App.tsx` 根据当前 URL 渲染对话页、Agent 管理页、Registry 管理页、Archive、Memory 或 Automation 页面。对话页继续由 layout 层读取全局状态并分发给 Composer、Timeline、Worker 列表和右侧调试面板。

## 边界与非目标
- 壳层只负责页面编排，不直接解释后端协议事件。
- 管理页路由只是前端入口，具体数据读写由 `src/shared/data/` 封装。
- 不在 layout 层堆业务状态归并逻辑；跨模块状态归并应进入 reducer、hooks 或对应 feature。
