# 附件上传与引用

## 当前状态
Composer 附件由 `useComposerAttachments`、`composerAttachments.ts` 和 `ComposerAttachments` 负责，上传接口为 `/api/upload`。附件发送时同时携带 `references` 和前端展示用 `attachments` 元数据。

## 核心职责
- 支持文件选择、拖拽、粘贴和 Desktop 截图转附件。
- 为图片创建本地 preview URL，并在清理时 revoke。
- 上传文件后保存 resource URL、references 和附件展示元数据。
- 在发送前阻止仍在 uploading 或 error 状态的附件。

## 核心流程
用户添加文件后生成 pending attachment，随后调用 `uploadComposerAttachments`。后端返回 chatId、references 和 resource URL 后，附件进入 ready 状态。发送消息时，ready attachments 的 references 进入 query payload，attachments 元数据用于用户消息和回放展示。

## 边界与非目标
- 附件上传不等同于 artifact 发布；artifact 是运行中后端事件 `artifact.publish`。
- steering 活跃 run 时不支持携带附件。
- 前端不读取文件内容做协议解释，只负责上传、预览和发送引用。
