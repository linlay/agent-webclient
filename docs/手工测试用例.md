# 手工测试用例

## 当前状态
本项目以 Jest 单元测试为主，核心联调依赖外部 AGENT API、语音服务、Desktop 宿主和浏览器能力。手工测试用于覆盖代理、实时流、HITL、附件、Viewport、语音和发布包结构等跨系统场景。

## 核心职责
- 给出本地开发、容器部署和 release 包的最小检查路径。
- 覆盖 SSE/WS、attach、submit、steer、interrupt 等运行控制。
- 覆盖附件上传、artifact resource、viewport iframe、HITL 四种 mode。
- 明确哪些失败应回到后端协议或部署配置排查。

## 核心流程
先准备 `.env` 并启动 `make dev` 或容器。确认页面可打开、agents/chats 可加载，再执行一次普通 query、一次运行中 steer、一次 interrupt、一次带附件 query。随后用后端可触发的样例验证 artifact.publish、viewport block、frontend tool、question、approval、form、plan awaiting 和 voice。

## 边界与非目标
- 手工测试不替代 `make test`。
- 没有可用后端时，只能验证静态页面、路由和构建，无法完成实时协议回归。
- 语音和 Desktop 桥接需要对应运行环境，普通浏览器失败不一定代表前端代码错误。
