# 开发代理与生产反向代理

## 当前状态
本地开发由 Webpack Dev Server 代理 `/api/*`、`/ws` 和可选 `/api/voice/*`。容器部署由 Nginx 模板代理普通 API、主 WebSocket 和可选语音服务。Desktop Program Bundle 的 HTTP 托管由 ZenMind Desktop main process 实现。

## 核心职责
- 本地开发时将 `/api/*` 和 `/ws` 转发到 `BASE_URL`。
- 设置 `VOICE_BASE_URL` 时追加语音 HTTP 和 `/api/voice/ws` 代理。
- 对 SSE 与 WebSocket 禁用代理缓冲，避免实时事件延迟。
- 保持 Webpack HMR 使用 `/__webpack_hmr`，避免与业务 `/ws` 冲突。

## 核心流程
`make dev` 加载 `.env` 后启动 Webpack Dev Server；生产容器启动时 `nginx-entrypoint.sh` 将 `BASE_URL` 和可选 `VOICE_LOCATIONS` 注入 Nginx 配置；浏览器始终访问同源 `/api/*`、`/ws`、`/api/voice/*`，由托管层转发到上游服务。

## 边界与非目标
- 前端代理只负责转发，不修正后端协议语义。
- `VOICE_BASE_URL` 为空时语音代理和语音入口都应关闭。
- Program Bundle 不包含 Node.js 服务进程；不要把 dev server 行为写成 Desktop 运行时事实。
