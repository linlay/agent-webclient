# 请求路由缓存与鉴权错误

## 当前状态
`src/shared/data/routedClient.ts` 在 HTTP fallback 与 WebSocket transport request 之间路由请求，`serverState.ts` 提供轻量缓存，`platformError.ts` 负责错误归一化展示。访问 token 和 Desktop app token 相关逻辑也收敛在数据层。

## 核心职责
- 根据当前 transport mode 决定请求走 WebSocket route 还是 HTTP fallback。
- 对 GET 端点使用 TTL 和 dedupe 缓存，减少重复拉取。
- 在写操作后按端点前缀失效缓存。
- 统一处理 access token、app auth refresh 和平台错误展示。

## 核心流程
调用方使用 `routedClient` 暴露的函数；函数内部通过 `routeEndpoint` 解析 payload、选择 transport、应用缓存策略。失败时回落到 HTTP 或抛出归一化错误，UI 层只展示 `formatPlatformErrorForDisplay` 的结果。

## 边界与非目标
- 缓存只服务前端 server-state，不承担离线数据库职责。
- 鉴权 helper 只处理前端 token 携带和刷新，不定义后端权限模型。
- route transport 只用于适合路由的接口；resource、voice-ws、SSE 等特殊传输保持专门路径。
