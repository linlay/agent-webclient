# 版本化打包与部署

## 当前状态
仓库支持 Program Bundle 与 Image Bundle 两种发布形态。Program Bundle 面向 ZenMind Desktop 托管静态资源，Image Bundle 面向离线 Docker 部署。版本号来自根目录 `VERSION`，格式为 `vX.Y.Z`。

## 核心职责
- `make release` / `make release-program` 生成 Desktop Program Bundle。
- `make release-image` 生成包含镜像 tar、compose 和脚本的 Image Bundle。
- 保证产物只保留在 `dist/release/`，并带版本化文件名。
- 输出 manifest、README.txt、`.env.example`、frontend dist 和部署脚本。

## 核心流程
release 脚本读取版本号并准备隔离构建环境，生产构建前端静态资源，再按目标平台组装压缩包。Image release 会构建单架构 Docker 镜像并导出 tar，连同 release compose 和启动脚本一起打包。

## 边界与非目标
- Program Bundle 不包含后端入口，也不启动 Node.js 子进程。
- Image Bundle 只交付 webclient 静态代理容器，不包含 AGENT API 后端。
- 发布脚本使用 `.env` 或 `.env.example` 完成构建配置，但真实部署值由部署端 `.env` 决定。
