# 运行参数模型与访问级别

## 当前状态
Composer 的运行参数包含 access level、模型覆盖、reasoning effort、service tier 和 planning mode 等。控件主要由 `QuerySettingsControls` 承载，运行时 access level 变更通过 `/api/access-level` 类接口同步。

## 核心职责
- 展示并持久化当前会话或当前 agent 的模型选择。
- 为 CODER agent 加载模型选项并处理默认模型覆盖。
- 将 access level、model override 和控制参数随 query payload 发送。
- 在运行中支持访问级别调整，并同步前端 agents 状态。

## 核心流程
Composer 打开运行参数控件时加载模型选项。发送消息时，`buildControlsParams`、`modelOverride` 和 `accessLevel` 被传入发送事件，最终由 `buildQueryPayload` 写入 `/api/query` 请求。运行中调整 access level 时调用 data client 并更新本地 agent summary。

## 边界与非目标
- 前端只展示后端返回的模型与权限能力，不自行推断真实权限。
- 非 CODER agent 不展示 CODER 专属模型覆盖入口。
- 模型参数只作为请求提示，最终执行语义由后端决定。
