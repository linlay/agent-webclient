# 时间线事件处理与渲染

## 当前状态
Timeline 模块负责把 live/replay 事件转换为消息、content、reasoning、planning、tool、awaiting-answer、source 等节点，再由 `ConversationStage`、`TimelineRow` 和各类 block 组件渲染。

## 核心职责
- 根据 `event.type` 分派事件处理器并生成 `EventCommand`。
- 维护 node id 映射、文本增量、tool 状态、source、debug event 和本地 cache。
- 支持 live 与 replay 共用同一处理路径。
- 为右侧 debug panel 和 run transcript 提供可读输出。

## 核心流程
`useAgentEventHandler` 接收事件并调用 `processEvent`。处理函数生成命令后由 `eventDispatchHandlers` 转为 App action，更新 timeline node map 和 order。渲染层按 node kind 选择对应 block，并对 markdown、source、tool pill、awaiting answer 等做专门展示。

## 边界与非目标
- Timeline 不发起 query，也不提交 HITL；它只展示事件和派生状态。
- 事件字段兼容应收敛在 processor，不在每个展示组件重复读取原始事件。
- Debug 展示不代表协议规范，只是联调观察视图。
