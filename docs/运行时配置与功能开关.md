# 运行时配置与功能开关

## 当前状态
运行时配置由构建/托管层写入 `runtime-config.js`，浏览器端通过 `globalThis.__AGENT_WEBCLIENT_RUNTIME_CONFIG__` 读取。功能开关集中在 `src/shared/config/featureFlags.ts`，语音是否启用由 `VOICE_BASE_URL` 推导出的 `VOICE_ENABLED` 控制。

## 核心职责
- 读取 `.env` 或部署环境变量中的前端开关。
- 暴露 debug panel、delta logs、settings menu、quick actions、memory、voice 等开关。
- 在开发服务和容器入口中生成浏览器可读取的 runtime config。
- 避免把私有上游地址或 token 硬编码进业务代码。

## 核心流程
开发模式下，Webpack dev server 的 `/runtime-config.js` 动态返回配置脚本。容器模式下，`scripts/nginx-entrypoint.sh` 在启动时写入同名脚本。业务代码通过 `readRuntimeConfigValue` 和 feature flag helper 读取开关，再决定是否展示对应入口。

## 边界与非目标
- `.env.example` 是变量契约；真实 `.env` 不提交。
- 配置只控制前端入口与客户端行为，不替代后端能力探测。
- 不在 React 组件中直接读取 `process.env`。
